# Cascade Web PWA

A progressive web application for the Cascade daemon. Provides a browser-based interface for status monitoring, file browsing, and capability management.

## Roles

The PWA supports three authentication roles:

| Role | Description |
|------|-------------|
| `owner` | Full local access; the device owns the node. |
| `named-user` | A specific user on a shared device, scoped to the grants they hold. |
| `bearer` | A portable capability token presented to a remote node via `cascade remote --token`. |

Owner and named-user tokens carry a device ID. Bearer tokens carry the issuer device ID and capability scope from the signed token.

## Setup

```bash
pnpm install
```

## Development

```bash
pnpm dev
```

The dev server proxies `/api/*` requests to the daemon. Set `VITE_API_HOST` and `VITE_API_PORT` to point at a running daemon (default: `localhost:8080`).

## Build

```bash
pnpm build
```

Output is written to `apps/web/dist/`.

## Type checking

```bash
pnpm typecheck
```

## Authentication

Tokens are stored in `localStorage` and are only persisted when the page is served from a secure context (HTTPS or localhost). The token is a JSON object containing at minimum a `role` field (`owner`, `named-user`, or `bearer`).

Bearer tokens are verified by the remote node's signed-token machinery; the PWA merely carries the token structure produced by `cascade token issue`.

## API

The PWA communicates with the daemon over its HTTP API (enabled with `cascade start --web`). Endpoints:

| Method | Path | Description |
|--------|------|-------------|
| `GET` | `/status` | Node status, backend health, cache stats, peer list |
| `GET` | `/files?parent=<id>` | List directory entries |
| `GET` | `/files/<id>` | Single file entry |
| `POST` | `/files/<id>/pin` | Pin a file |
| `POST` | `/files/<id>/unpin` | Unpin a file |
| `POST` | `/cache/warm` | Warm cache for a path |
| `POST` | `/cache/evict?path=<path>` | Evict files from cache |
| `GET` | `/grants` | List grants held on this node |
| `POST` | `/grants` | Add a grant |
| `DELETE` | `/grants/<id>` | Revoke a grant |
| `GET` | `/shares?folder=<path>` | List directional shares |
| `POST` | `/shares` | Add a directional share |
| `DELETE` | `/shares` | Revoke a directional share |
| `GET` | `/tokens` | List issued tokens |
| `DELETE` | `/tokens/<id>` | Revoke a token |

## Service worker

The service worker is generated by Workbox via `vite-plugin-pwa`. API GET requests use a `NetworkFirst` strategy with a 5-second timeout; static assets use `CacheFirst`. The app works offline for cached content.
